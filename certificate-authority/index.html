The xANNA certificate authority

xANNA is a PKI certificate authority, but not so public.  On who's authority did xANNA become a CA?  Answer: xANNA's.

The motivating purpose is that sometimes it's useful to set up authentication systems without having to be hooked into the public key infrastructure. This is true even these days where dynamic certificate generation is around.  Examples include private networks, testing, and so on.

When others are involved (e.g. open source collaboration or intruders!), having a degree of security is good idea.  So, just copying around credentials soon becomes problematic.  Self-signed cert's just generate more and more work.  Testing when you have to disable security leaves big and dangerous holes in the testing.

So, the xANNA CA is a real CA. The xANNA root and intermediate certificates are signed by the xANNA registrar.  We publish signature information here in the xANNA.  You can also get the certificate from alternate channels.  For example, you can stop by the registrar's desk.

The xANNA root CA certificate, the xANNA SSL CA certificate, as well as other certificates issued by xANNA can be found in the ca registry.
